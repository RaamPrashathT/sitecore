# Sitecore Design Context

## Design Context

### Users
Construction organization admins, engineers, and clients. Admins manage everything — procurement, materials, budgets, team. Engineers and clients are read-only observers. Used primarily at a desk, in an office or site office context. The interface needs to handle dense data without feeling overwhelming.

### Brand Personality
Efficient, professional, trustworthy. The product is about: Progress, Professional, Money, Management. It should feel like a serious tool — not a startup SaaS, not a consumer app. Think: a well-made instrument, not a toy.

### Aesthetic Direction
- Light mode only
- Primary accent: green (`oklch(0.55 0.18 145)`) — locked in, used consistently
- Neutrals: white surfaces, slate/stone for backgrounds and secondary text
- Typography: currently Inter everywhere — needs a more distinctive pairing
- No dark mode
- No animations except functional state transitions (loading, open/close)
- No decorative elements — every visual element must earn its place
- Anti-references: generic AI dashboards, random spacing, inconsistent hierarchy, gratuitous animation

### Design Principles
1. **Hierarchy through density** — data-dense but scannable; use size, weight, and spacing contrast deliberately, not uniformly
2. **Green means action or success** — the accent is reserved for interactive elements, active states, and positive status; never decorative
3. **Mono for numbers, sans for labels** — IBM Plex Mono for all numeric values, quantities, codes; body font for everything else
4. **Progressive disclosure** — show only what's needed at each level; table → drawer → full page
5. **No noise** — borders, shadows, and backgrounds only when they create necessary separation; flat where possible
